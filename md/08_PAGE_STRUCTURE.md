# 08 — Page Structure

## Recommended architecture

Use a single-page cinematic homepage with optional project detail pages.

Routes:

```text
/
 /work/sws-luxury
 /work/ghost-engineer
 /work/obsidian-finish
 /work/iron-forge
```

If speed matters, build only `/` first and link directly to live projects.

## Homepage section order

### 1. Loader

Purpose:

- create entry
- establish brand tone

Content:

```text
GURTEJ
DIGITAL EXPERIENCES
```

### 2. Hero

Purpose:

- immediate visual identity

Content:

```text
GURTEJ
Creative developer crafting premium digital experiences through motion, visual systems, and sharp execution.
```

CTAs:

- Selected Work
- Contact

### 3. Identity Strip

Purpose:

- quick positioning after hero

Content:

```text
Creative Developer / Motion-led Interfaces / Premium Landing Pages / Digital Product Experiences
```

Motion:

- horizontal marquee or subtle reveal
- no fast ticker

### 4. Manifesto / Principles

Purpose:

- communicate thinking

Content:

```text
Motion is not decoration.
Luxury is restraint.
Interfaces should communicate before users think.
A website should not just load. It should arrive.
```

### 5. Selected Work

Purpose:

- prove ability

Projects:

1. SWS Luxury
2. Ghost Engineer
3. Obsidian Finish Studio
4. Iron Forge Athletics

Layout:

- one project per large panel
- index numbers
- live CTA
- project preview image

### 6. Process

Purpose:

- show that work has structure, not just visuals

Steps:

1. Position the brand
2. Build the visual system
3. Structure the conversion path
4. Add motion with purpose
5. Optimize for mobile

### 7. Visual Experiments

Purpose:

- show creative potential

Content:

- motion studies
- 3D/typography experiments
- image treatments
- interface concepts

If assets are not ready, create placeholder editorial cards.

### 8. About

Purpose:

- background and credibility

Include:

- B.Tech student
- web development/software engineering focus
- responsive business websites
- visual systems and problem-solving
- certificates optionally

Do not make this too long.

### 9. Contact

Purpose:

- conversion

Content:

```text
Let’s build something with presence.

gurtejbjr.29107@gmail.com
Amritsar, Punjab
```

Optional:

- phone
- social links
- contact form

### 10. Footer

Purpose:

- closure

Content:

```text
Designed and built by Gurtej.
Digital experiences with motion, clarity, and restraint.
```

## Project detail page structure

### Hero

- project title
- category
- live link
- large screenshot/video

### Overview

- one paragraph
- role
- year
- tools/stack

### Challenge

- what problem the website solved

### Direction

- design and content strategy

### Key Screens

- homepage
- mobile
- section detail

### Craft

- motion
- typography
- layout
- responsiveness

### Result / Learning

- what the project proves

### Next Project

- cinematic transition to next case study

## Navigation behavior

Desktop:

```text
GURTEJ     Work / Principles / About / Contact
```

Mobile:

```text
GURTEJ     Menu
```

Nav should appear over hero but stay subtle.

On scroll:

- reduce nav height
- add blurred dark background
- keep readable

## Scroll behavior

Use smooth scroll.

Suggested section rhythm:

- Hero: fullscreen
- Manifesto: pinned text sequence
- Work: long scroll/pinned project panels
- Process: structured grid
- About: image + text
- Contact: fullscreen close

## Archive behavior

Iron Forge can be an archive tile below featured projects.

Do not bury it completely, but do not give it the same drama as SWS Luxury or Ghost Engineer.

## Information hierarchy

Most important:

1. Hero
2. SWS Luxury
3. Ghost Engineer
4. Contact
5. About

Least important:

- certificates
- education
- skill list

Those can exist, but they should not dominate.

## Sections to avoid

Do not include:

- skill percentage bars
- rotating tech icons
- generic services cards
- huge education timeline
- fake testimonials
- “my hobbies”
- unrelated personal facts
