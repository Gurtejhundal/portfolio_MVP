# 06 — Project Case Studies

## Project hierarchy

The portfolio should not give every project equal weight.

Recommended order:

1. **SWS Luxury / Luxury Taste** — strongest visual/commercial premium project
2. **Ghost Engineer** — strongest product/system concept
3. **Obsidian Finish Studio** — polished service business website
4. **Iron Forge Athletics** — archive/supporting commercial build

## Case study format

Each project should follow this structure:

1. Title
2. Category
3. One-line thesis
4. Role
5. Challenge
6. Direction
7. Key features
8. Visual/system decisions
9. Live link

Keep it concise. The homepage should not become a blog.

## Project 01 — SWS Luxury

### Category

Luxury Commerce Experience

### Source link

https://luxury-taste.vercel.app/

### One-line thesis

A premium commerce flagship for attars, perfumes, jewelry, and concierge gifting.

### Role

- design direction
- frontend implementation
- responsive layout
- product storytelling
- cart / WhatsApp order flow presentation

### Challenge

Luxury products cannot be presented like a crowded catalogue. The site needed to feel curated, restrained, and service-led while still making purchasing/enquiry easy.

### Direction

Build the experience like a private appointment:

- fewer products shown better
- strong hero statement
- curated collection rooms
- concierge ordering
- gift-driven storytelling
- premium wrapping and occasion context

### Key features

- luxury hero section
- category navigation
- curated product cards
- gifting edition section
- WhatsApp concierge CTA
- trust notes
- client signals/testimonials
- cart-to-message ordering structure

### Portfolio card copy

```text
SWS Luxury

A premium commerce flagship for attars, perfumes, jewelry, and concierge gifting.

Built around restraint, curated collections, WhatsApp ordering, and high-end product storytelling.
```

### Visual treatment in portfolio

Use:

- dark luxury mockup
- product-style preview
- gold/ivory accents
- slow horizontal reveal

## Project 02 — Ghost Engineer

### Category

AI Product / Open-Source System

### Source link

https://ghost-engineer-psi.vercel.app/

### One-line thesis

An open-source impact lab that turns real-world problems into buildable public-good blueprints.

### Role

- product structure
- interface design
- frontend build
- AI workflow presentation
- contributor system framing

### Challenge

Most idea generators produce vague outputs. Ghost Engineer needed to feel like a system that turns social problems into practical, buildable, pilot-ready packages.

### Direction

The product was structured around:

- problem-first workflow
- impact blueprint
- contributor board
- pilot launch plan
- multi-agent review system
- GitHub-ready file outputs

### Key features

- problem input flow
- blueprint generation concept
- impact scoring
- contributor roles
- four-agent council
- GitHub pack
- fallback demo readiness
- developer hub

### Portfolio card copy

```text
Ghost Engineer

An open-source impact lab that turns real-world problems into buildable public-good blueprints.

Structured around problem-first workflows, agent review, contributor tasks, GitHub-ready files, and pilot planning.
```

### Visual treatment in portfolio

Use:

- technical dark system style
- grid lines
- blueprint feel
- agent cards
- data/system motion

## Project 03 — Obsidian Finish Studio

### Category

Premium Service Website

### Source link

https://car-detailing-studio-seven.vercel.app/

### One-line thesis

A conversion-focused website for a premium car detailing and protection studio.

### Role

- brand positioning
- landing architecture
- frontend build
- service conversion flow
- responsive business website

### Challenge

Car detailing services can easily feel generic. The site needed to communicate trust, finish quality, premium care, and booking intent quickly.

### Direction

Build around:

- premium detailing bay identity
- protection/correction/detailing services
- trust-led service explanation
- package clarity
- results and gallery
- booking path

### Key features

- hero with clear service positioning
- core services
- stats/trust indicators
- package sections
- inspection-led CTA
- gallery/result proof
- contact and booking flows

### Portfolio card copy

```text
Obsidian Finish Studio

A premium car detailing and protection website built around finish quality, service clarity, and booking intent.
```

### Visual treatment in portfolio

Use:

- black/graphite panels
- automotive reflections
- sharp highlights
- metallic transitions

## Project 04 — Iron Forge Athletics

### Category

Fitness / Commercial Website

### Source link

https://gym-two-sigma.vercel.app/

### One-line thesis

A premium gym website built around coaching credibility, clear membership paths, and conversion.

### Role

- landing strategy
- responsive build
- service explanation
- CTA structure
- commercial content architecture

### Challenge

Gym websites often rely on loud visuals and weak promises. This project needed to feel serious, credible, and commercially clear.

### Direction

Build around:

- strength-first coaching
- premium equipment
- disciplined programming
- beginner and professional clarity
- visible next steps
- trial conversion

### Key features

- premium gym hero
- membership plans
- programs
- trainer profiles
- schedule preview
- proof/results section
- blog/education content
- FAQ and trial CTA

### Portfolio card copy

```text
Iron Forge Athletics

A premium gym website structured around coaching credibility, clear membership paths, and conversion.
```

### Visual treatment in portfolio

Use as archive/supporting project.

Do not let it dominate the portfolio. It is useful proof of commercial structure, but Luxury and Ghost Engineer are stronger identity projects.

## Homepage work section rules

- Do not show tiny project cards only.
- Give each project room.
- Use large title and preview.
- Use index numbers.
- Add live link.
- Keep body copy short.
- Make transitions feel like chapters.

## Project labels

Use labels like:

- Luxury Commerce
- AI Product System
- Premium Service Website
- Commercial Fitness Website

Avoid labels like:

- React Project
- HTML CSS Project
- College Project
