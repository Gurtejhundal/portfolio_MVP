# Award Portfolio MD Pack

This pack is a complete planning/specification system for building Gurtej's portfolio website in Codex.

The site direction is not a normal resume portfolio. It should feel like a cinematic personal brand website for a creative developer building premium digital experiences.

## Files

1. `00_MASTER_BRIEF.md` — core vision, positioning, non-negotiables
2. `01_SRS.md` — software requirements specification
3. `02_DESIGN_SYSTEM.md` — visual language, colors, typography, spacing, art direction
4. `03_CONTENT_STRATEGY.md` — final website copy, tone, section narratives
5. `04_HERO_SECTION.md` — exact hero layout and behavior
6. `05_ANIMATION_CHOREOGRAPHY.md` — motion system, GSAP/scroll rhythm, transitions
7. `06_PROJECT_CASE_STUDIES.md` — project hierarchy and case-study writing
8. `07_IMAGE_ASSET_GUIDE.md` — required personal images, editing, naming, usage
9. `08_PAGE_STRUCTURE.md` — routes, sections, information architecture
10. `09_COMPONENTS.md` — component-level requirements
11. `10_RESPONSIVE_MOBILE.md` — mobile-first adaptation rules
12. `11_SEO_PERFORMANCE_ACCESSIBILITY.md` — performance, SEO, accessibility, image rules
13. `12_CODEX_BUILD_PROMPT.md` — final hard prompt to paste into Codex
14. `13_CONTENT_INVENTORY.md` — personal/CV information and project source inventory
15. `14_REFERENCE_DIRECTION.md` — inspiration model and what to copy/not copy

## Build philosophy

Do not build a decorative portfolio. Build a controlled cinematic experience.

The target perception:

> Creative developer. Premium digital experiences. Cinematic interface systems. Commercially useful execution.

The mistake to avoid:

> Over-animated student portfolio with random effects and weak content.

Every effect must support hierarchy, pacing, or identity.
